Developer Experience
A clean API for
the gate decision
REST or MCP. Rust under the hood. Bring your own wallet or provision one through Agentra. The endpoint that matters is POST /v1/authorizations -- everything else exists to make that one call accurate.
Rust + Axum PostgreSQL Non-Custodial MCP + REST Turnkey WaaS
gate.sh 
# 1. Register an agent
curl -X POST https://api.agentrapay.ai/agents \
  -H "Content-Type: application/json" \
  -d '{"name": "trading-bot", "purpose": "payments",
       "operator": "My Co"}'

# Returns: id, api_key (agn_...), trust_tier: "Unverified"

# 2. Verify identity (KYA)
curl -X POST https://api.agentrapay.ai/agents/{id}/kya \
  -H "X-API-Key: agn_..." \
  -H "Content-Type: application/json" \
  -d '{"field": "operator_name", "value": "My Co"}'

# trust_tier: "Basic", single_tx: $1,000, daily: $5,000

# 3. Call the gate before every transaction
curl -X POST https://api.agentrapay.ai/v1/authorizations \
  -H "X-API-Key: agn_..." \
  -H "Content-Type: application/json" \
  -d '{"sender_agent_id": "...", "receiver_address": "0x...",
    "amount_usdc": 10.0, "purpose": "api-payment"}'

# Returns:
#   decision: "approve" | "deny" | "escalate"
#   authorization_id, attestation, unsigned_tx

# 4. Settle on your rail (x402, MPP, direct USDC, etc.)
#    Attach authorization_id for audit. Done.

Positioning
A gate, not a rail
Clarity matters when the agentic-payments stack has a dozen overlapping protocols. Agentra is what gets called before the money moves. We work with everyone else, not against them.
  • ✕
    Not a payment processor
    Agentra never custodies funds, never settles, never touches private keys.
  • ✕
    Not a wallet
    Use Turnkey, Privy, Crossmint, or your own. Agentra stores the address, not the key.
  • ✕
    Not a replacement for x402, MPP, or Skyfire
    Those are settlement rails. Agentra is the authorization step that fires immediately before them.
  • ✕
    Not KYC for humans
    Agentra verifies agents. Identity proofing of human operators happens elsewhere in your stack.
