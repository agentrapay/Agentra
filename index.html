How It Works
From zero to authorized
in four API calls
An AI agent goes from unknown to fully verified -- with identity, reputation, and authorization capability -- in under 60 seconds. No human in the loop.
1
Register Agent
POST /agents with name, purpose, and operator. Receive an API key (agn_ prefix) and agent ID instantly.
2
Verify Identity (KYA)
POST /agents/{id}/kya with operator attestation. Trust tier upgrades immediately. Higher tiers unlock higher transaction limits.
3
Check Counterparty
GET /v1/trust/{agent_id} to check the other party's reputation before transacting. Trust tier, score band, and mandate limits returned.
4
Authorize Transaction
POST /v1/authorizations. Agentra returns a signed attestation. The agent signs the transaction with its own key and submits to the chain.

quickstart.sh 
# 1. Register an agent
curl -X POST https://api.agentrapay.ai/agents \
  -H "Content-Type: application/json" \
  -d '{"name": "trading-bot", "purpose": "payments", "operator": "My Co"}'
 
# Returns: id, api_key (agn_...), trust_tier: "Unverified"
 
# 2. Verify identity (KYA)
curl -X POST https://api.agentrapay.ai/agents/{id}/kya \
  -H "X-API-Key: agn_..." \
  -H "Content-Type: application/json" \
  -d '{"field": "operator_name", "value": "My Co"}'
 
# trust_tier: "Basic", single_tx: $1,000, daily: $5,000
 
# 3. Check counterparty trust before paying
curl https://api.agentrapay.ai/v1/trust/{counterparty_id} \
  -H "X-API-Key: ask_..."
 
# trust_tier: "Standard", score_band: "green"
 
# 4. Authorize a transaction
curl -X POST https://api.agentrapay.ai/v1/authorizations \
  -H "X-API-Key: agn_..." \
  -H "Content-Type: application/json" \
  -d '{"sender_agent_id": "...", "receiver_address": "0x...",
    "amount_usdc": 10.0, "purpose": "api-payment"}'
 
# Returns: authorization_id, attestation, unsigned_tx
