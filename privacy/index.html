Privacy Policy

Effective: March 17, 2026 | Last updated: March 17, 2026

This Privacy Policy describes how ProTech Data Security Services LLC d/b/a Agentra ("Agentra," "we," "us," or "our") collects, uses, and shares information when you use the Agentra API, website, and related services (the "Service").

1. Information We Collect

Agent Registration Data. When you register an agent via the API, we collect the agent name, purpose, and operator name you provide. These fields are required and stored in our database.

KYA Verification Data. When you submit Know Your Agent verification, we collect the identity fields you provide (agent name, purpose, operator name, operator contact, attestations). This data is used to calculate your agent's trust tier.

API Keys. We generate and issue API keys at registration. We store a one-way cryptographic hash of each key. We do not store the plaintext API key after it is shown to you once.

Transaction Data. We record authorization requests, settlement reports, trust score lookups, and referral events. This includes agent IDs, counterparty addresses, transaction amounts, timestamps, and status.

Wallet References. When you provision a wallet, we store a reference to the wallet (external wallet ID, wallet address, provider name). We do not store private keys. Private keys are held by the third-party wallet provider (currently Turnkey) and are accessible only to you.

Usage Data. We log API endpoint usage, request timestamps, and IP addresses for rate limiting, abuse prevention, and service monitoring.

Blockchain Data. Authorization attestations and trust tier records may be published to public blockchains (currently Base Sepolia testnet) via the Ethereum Attestation Service (EAS). On-chain data is publicly visible and cannot be deleted.

2. How We Use Your Information

We use the information we collect to:

  • Provide, operate, and maintain the Service
  • Calculate and update trust tiers and reputation scores
  • Process authorization requests and record settlements
  • Track referrals and calculate fee reductions
  • Enforce rate limits and usage quotas
  • Detect and prevent abuse, fraud, and unauthorized access
  • Generate aggregate, anonymized analytics (e.g., network stats, performance metrics)
  • Communicate with you about the Service (if you provide contact information)
  • Comply with legal obligations

3. Information We Share

We do not sell your data.

We may share information in the following circumstances:

  • Public endpoints. Certain data is available through public API endpoints by design. Trust scores (via GET /v1/trust/{agent_id}), network stats (via GET /v1/network/stats), performance metrics (via GET /v1/status/metrics), and on-chain attestations are publicly accessible. Agent names and reputation scores may be visible to counterparties during transactions.
  • On-chain attestations. EAS attestations published to Base are permanently public. These include agent ID (as bytes32), trust tier, reputation score, reputation zone, and transaction count. On-chain data cannot be modified or deleted.
  • Service providers. We use third-party infrastructure providers including Railway (hosting), Turnkey (wallet provisioning), and Cloudflare (DNS and CDN). These providers process data as necessary to deliver their services.
  • Legal requirements. We may disclose information if required by law, subpoena, court order, or government request.
  • Business transfers. In connection with a merger, acquisition, or sale of assets, your data may be transferred to the acquiring entity.

4. Data Retention

We retain agent registration data, transaction records, and reputation history for the lifetime of the Service. This is necessary because reputation scoring depends on historical transaction data, and on-chain attestations reference agent records.

API usage logs are retained for 90 days. IP addresses in access logs are retained for 30 days.

If you request deletion of your agent account, we will delete your registration data and API key hash. Transaction records will be anonymized (agent ID removed). On-chain attestations cannot be deleted.

5. Data Security

We implement industry-standard security measures including:

  • API keys stored as one-way cryptographic hashes (never in plaintext)
  • Encrypted connections (TLS) for all API traffic
  • Database access restricted to the application layer
  • Non-custodial architecture (we never hold private keys)

No system is perfectly secure. We cannot guarantee that unauthorized access, data breach, or loss will never occur.

6. Your Rights

Depending on your jurisdiction, you may have rights regarding your personal data, including:

  • Access. Request a copy of the data we hold about your agent(s).
  • Correction. Request correction of inaccurate data via KYA resubmission.
  • Deletion. Request deletion of your agent account (subject to the limitations in Section 4).
  • Portability. Request your transaction history in a machine-readable format.

To exercise these rights, email support@agentrapay.ai.

7. California Residents (CCPA)

If you are a California resident, you have the right to know what personal information we collect, request its deletion, and opt out of the sale of personal information. We do not sell personal information. To submit a request, email support@agentrapay.ai.

8. International Users

The Service is operated from the United States. If you access the Service from outside the United States, your data will be transferred to and processed in the United States. By using the Service, you consent to this transfer.

9. Children

The Service is not directed to individuals under 18. We do not knowingly collect personal information from children. If we learn that we have collected data from a child under 18, we will delete it.

10. Cookies and Tracking

The agentrapay.ai website does not use cookies or third-party analytics trackers. The API does not set cookies. We may use server-side logging for operational monitoring.

11. Changes to This Policy

We may update this Privacy Policy at any time. Changes take effect when posted. We encourage you to review this page periodically. Material changes will be communicated via the API health endpoint or email if provided.
