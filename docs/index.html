Getting Started

Onboard your AI agent in under 60 seconds. No credit card, no approval process, no human in the loop.

What is Agentra?

Agentra is trust and compliance infrastructure for autonomous AI agents. When your agent needs to transact with another agent or pay for a service, Agentra handles the hard parts: identity verification, counterparty reputation checks, spending mandate enforcement, and transaction authorization.

Agentra is non-custodial. Your agent owns its private keys. Agentra authorizes transactions and provides trust attestations, but never holds, moves, or has access to your agent's funds.

Agentra works alongside payment protocols like MPP (Stripe's Machine Payments Protocol) and x402 (Coinbase). Those protocols solve how agents pay. Agentra solves whether the agent should be trusted to transact and whether the counterparty is safe to receive funds. If MPP is the credit card network, Agentra is the credit bureau.

Prerequisites

You need curl (or any HTTP client) and a crypto wallet address if you want to transact on-chain. That's it. The API is at https://api.agentrapay.ai. All requests use JSON. Authentication is via the X-API-Key header. Agentra is the trust layer; you bring your own wallet and payment rails.

STEP 1 Register Your Agent ~2 seconds

# Register a new agent
curl -X POST https://api.agentrapay.ai/agents \
  -H "Content-Type: application/json" \
  -d '{
    "name": "my-trading-agent",
    "purpose": "automated trading",
    "operator": "My Company Inc."
  }'

Response:

{ "id": "a1b2c3d4-...", "name": "my-trading-agent", "wallet_address": "0x...", "trust_tier": "Unverified", "api_key": "agn_7f8a9b2c...", "message": "Save your API key now -- it will not be shown again." }
Save your API key immediately. It starts with agn_ and is only shown once. Use it in the X-API-Key header for all authenticated requests.

STEP 2 Verify Identity (KYA) ~2 seconds

Know Your Agent (KYA) verification upgrades your trust tier, unlocking higher transaction limits.

# Submit KYA verification
curl -X POST https://api.agentrapay.ai/agents/{your_agent_id}/kya \
  -H "X-API-Key: agn_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "agent_name": "my-trading-agent",
    "purpose": "automated trading",
    "operator_name": "My Company Inc."
  }'

Response:

{ "agent_id": "a1b2c3d4-...", "trust_tier": "Basic", "transfer_limit": 1000.0, "daily_limit": 5000.0, "verifications": [...] }

STEP 3 Provision a Wallet ~5 seconds

Agentra provisions a non-custodial wallet via Turnkey secure enclaves. Your agent owns the private key.

# Provision a Turnkey wallet
curl -X POST https://api.agentrapay.ai/v1/wallets/provision \
  -H "X-API-Key: agn_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"agent_id": "your_agent_id"}'
Non-custodial: Agentra stores the wallet reference but never has access to the private key. Your agent signs all transactions.

STEP 4 Authorize a Transaction ~3 seconds

Request authorization for a payment. Agentra verifies identity, checks counterparty reputation, enforces your mandate, and returns a signed attestation with an unsigned transaction.

# Request transaction authorization
curl -X POST https://api.agentrapay.ai/v1/authorizations \
  -H "X-API-Key: agn_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "agent_id": "your_agent_id",
    "to_address": "0xRecipientAddress...",
    "amount_usd": 10.00,
    "asset": "USDC"
  }'

Response includes:

{ "authorization_id": "uuid-...", "status": "approved", "attestation_sig": "0x...", "unsigned_tx": { ... }, "settle_url": "/v1/authorizations/{id}/settle" }

STEP 5 Settle ~2 seconds

After your agent signs and submits the transaction on-chain, report the settlement:

# Report settlement
curl -X POST https://api.agentrapay.ai/v1/authorizations/{authorization_id}/settle \
  -H "X-API-Key: agn_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"tx_hash": "0xYourTransactionHash..."}'

Using Agentra with MPP or x402

If your agent uses Stripe's Machine Payments Protocol (MPP) or Coinbase's x402 for payments, Agentra slots in as the pre-transaction trust check. The flow:

# 1. Your agent hits a service and gets an MPP/x402 payment challenge
# 2. Before paying, check the counterparty's reputation on Agentra
curl https://api.agentrapay.ai/v1/trust/{service_agent_id} \
  -H "X-API-Key: ask_your_service_key"
 
# 3. If the counterparty is green-zone, authorize the transaction
curl -X POST https://api.agentrapay.ai/v1/authorizations \
  -H "X-API-Key: agn_your_key" \
  -H "Content-Type: application/json" \
  -d '{"agent_id":"...","to_address":"...","amount_usd":10.0,"asset":"USDC"}'
 
# 4. Complete the MPP/x402 payment using your own wallet
# 5. Report settlement to Agentra to build your transaction history
Agentra does not replace your payment rails. It adds the trust and identity layer on top. Use MPP, x402, or any payment method your agent prefers. Agentra verifies both sides of the transaction and builds portable reputation over time.

Check Counterparty Trust

Before transacting with another agent, check their reputation:

# Check counterparty trust score
curl https://api.agentrapay.ai/v1/trust/{counterparty_agent_id} \
  -H "X-API-Key: ask_your_service_key"

Returns reputation score (0-100), trust tier, risk zone (green/yellow/red), and recommended maximum transaction value.

Referral Economics

When your agent's mandate includes require_counterparty_kya: true, any agent that onboards to transact with you is attributed as your referral. Each referral reduces your per-authorization fee by 1 basis point (cumulative, capped at 5 bps off the 15 bps base).

# Check your referral stats
curl https://api.agentrapay.ai/v1/agents/{your_agent_id}/referrals \
  -H "X-API-Key: agn_your_key_here"

Error Handling

All errors return JSON with an error field. Common status codes:

StatusMeaningAction
401Missing or invalid API keyCheck your X-API-Key header
402Payment required (x402)Authorize payment via /v1/authorizations
403Tier too low or counterparty riskComplete KYA or check counterparty
404Agent or resource not foundVerify the ID in your request
409Conflict (e.g., already referred)Resource already exists
402 responses include discovery headers. If your agent encounters a 402 from any service using Agentra, check the Link and X-Agent-Onboard headers for onboarding instructions.
